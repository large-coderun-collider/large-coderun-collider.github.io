Fill 0004 — Bulkhead & Bounded Queue

A compact, testable implementation of the Bulkhead concurrency pattern with load shedding.

Code: lcrc-experiments/fills/0004-bulkhead-queue

Accelerator

When a system experiences a massive traffic spike or slow downstream processing, unbounded concurrency will quickly deplete system memory, thread pools, and file descriptors. A Bulkhead isolates resources and enforces strict concurrency limits. By bounding both active execution slots and the waiting queue, the system gracefully sheds excess load (fail-fast rejection) instead of crashing from resource starvation.

MAPS

  • Concurrency Gate:
    • ACCEPTED: slot acquired immediately; task executes.
    • QUEUED: all active slots full; task waits in bounded queue.
    • REJECTED: both active slots and queue are saturated (Load Shedding).
  • State variables: active_count (currently executing) and queue_count (waiting in line).
  • Parameters: max_concurrent and max_queue.

BOB

References and Concepts:

  • Resource isolation prevents localized slowness from cascading into total host failure.
  • Bounded queues prevent Out-Of-Memory (OOM) crashes under sustained load spikes.
  • Load Shedding (rejecting excess traffic instantly) maintains predictable latency for accepted tasks.

CMS

Simulate a sequence of acquisitions and releases:

lcrc-bulkhead simulate --max-concurrent 2 --max-queue 1 --seq acquire,acquire,acquire,acquire,release,release --json

LCRCb

  • Zero external dependencies (pure Python standard library).
  • Automatic queue promotion upon slot release.
  • Clear separation between non-blocking admission checking (try_acquire) and task execution (submit).
